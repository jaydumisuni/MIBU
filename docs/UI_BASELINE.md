# Locked UI Baseline

The first mockups are now the locked visual baseline for MIBU.

## Rule

When building the real app, the design should stay as close as possible to the approved mockups:

- dark background
- neon blue / purple / orange accent colors
- MIBU icon and THETECHGUY TOOL breadcrumb
- large ghost hero artwork
- compact, polished, professional layout

## Android app baseline

### Screen 1: Welcome / import screen

Must include:

- MIBU title and THETECHGUY TOOL line
- large ghost hero image
- primary import button
- secondary actions for status and guide
- footer: By the THETECHGUY TOOL team

### Screen 2: Dashboard / waiting screen

Must include:

- account/session status cards
- Beijing target time card
- local target time card
- countdown section
- mobile-data reminder
- foreground-service status row
- primary waiting action
- logs button
- instructions button

## PC helper baseline

The PC app should be a small centered window, not a giant app.

### Screen 1: Setup steps

Must include:

- MIBU PC Helper title
- sidebar with brand logo / progress / helper card
- step-by-step setup list
- compact wizard feel
- THETECHGUY TOOL breadcrumb / footer

### Screen 2: Ready / status screen

Must include:

- compact centered layout
- connected device, ADB, APK install, and readiness rows
- Beijing time and local time panel
- recommendation panel
- bottom actions such as Install APK, Send to Phone, Open Guide, Next

## Build rule

Do not redesign the look casually. Improve code and behavior, but preserve the approved visual identity unless a later mockup replaces it.
